# Marine litter analysis — Arraial do Cabo

Reproducible reanalysis of marine-litter data collected at Praia Brava and
Ilha do Pontal, Arraial do Cabo, Rio de Janeiro, Brazil.

## Official data source

The analysis uses `data/raw/planilha_organizada.xlsx` as its official and
immutable input. All processed datasets, tables, figures, and statistical
results are generated from that file by scripts in this repository.

## Run in PowerShell

Open this folder in VS Code, start an integrated PowerShell terminal, and run:

```powershell
Set-ExecutionPolicy -Scope Process Bypass
.\run_analysis.ps1 -Stage validate
```

To reproduce every analysis:

```powershell
.\run_analysis.ps1 -Stage all
```

In VS Code, the same commands are available through **Terminal → Run Task**:

- `Validate official data`
- `Run complete analysis`

## Project structure

- `data/raw/`: immutable source data.
- `data/processed/`: standardized data generated by scripts.
- `scripts/`: validation, analysis, and figure-generation code.
- `outputs/tables/`: publication-ready tables.
- `outputs/figures/`: publication-ready figures.
- `reports/`: statistical and reproducibility reports.
- `docs/`: data dictionary and analytical decisions.
- `notebooks/`: legacy notebook retained for provenance; not used by the new
  workflow.

## Current analytical workflow

- Integrity validation and explicit Portuguese-to-English variable mapping.
- Descriptive summaries retaining all 24 transects.
- Composition tables by beach and season.
- Hellinger PCA for compositional structure.
- Exploratory K-means comparison using silhouette scores.
- Bray–Curtis PERMANOVA with 9,999 permutations.
- PERMDISP diagnostics before interpreting PERMANOVA.
- Publication figures exported as PNG and PDF.

## Reproducibility rule

Never edit files in `data/raw/`. If the source workbook changes, replace it
deliberately, record the new SHA-256 hash, and rerun the complete workflow.
