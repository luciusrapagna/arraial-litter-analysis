# GitHub and Zenodo preparation

## Proposed repository

- Repository name: `arraial-litter-analysis`
- Visibility: public, required for the standard GitHub–Zenodo workflow
- Suggested description: Reproducible analysis of marine-litter abundance and
  composition at Praia Brava and Ilha do Pontal, Arraial do Cabo, Brazil.

## Metadata required before the first release

For every creator:

- Full name as it should appear in the citation.
- Family name and given name.
- Institutional affiliation.
- ORCID, when available.
- Confirmed creator order.

Project-level decisions:

- Confirm the final repository title.
- Confirm whether the official workbook may be published openly.
- Select the license for data and code.
- Confirm the desired keywords.
- Confirm whether the associated manuscript has a DOI.

## Recommended release workflow

1. Review the official workbook and generated results.
2. Complete `CITATION.cff` and `.zenodo.json`.
3. Push the reviewed project to the public GitHub repository.
4. Enable the repository in Zenodo.
5. Create GitHub release `v1.0.0`.
6. Verify the archived files and metadata in the Zenodo draft.
7. Publish the Zenodo record and add its DOI badge to `README.md`.
